# universal-province-engine

> **法はコードに刻まれ、歴史は永久に残る。**
> 運営のいない世界で、あなたの武将は今日も生きている。

---

## これは何か

`universal-province-engine` は、**任意の地形データを受け取り、自律的に駆動する地域サーバーエンジン**です。

JSONファイルを差し替えるだけで、戦国日本にも、中世ヨーロッパにも、地球まるごとにも対応します。

最初の舞台は**日本・令制国68国**です。

---

## 四つのリポジトリが、ひとつの宇宙を作る

```
universal-asset-registry    資産の定義      （お宝・刀・茶器・城の証書）
Universal-Soul-Logic        魂の定義        （武将・老化・AP・婚姻・継承）
hex-shogun-map              地形の定義      （国境・ヘックスグリッド・地勢）
universal-province-engine   宇宙の定義      （これ）
```

最初の三つは「静的な定義」です。
このリポジトリは、それらが**相互作用する空間・時間・法**を定義します。

## 設計哲学

### 法はコード

ゲームのルールはスマートコントラクトとコードに封印されます。運営であっても変更できません。

サーバーが返すのは**事実のみ**です。数値とフラグだけ。文章はゼロ。

```
サーバーが返すもの（事実・不変）
{
  "province": "伊豆",
  "year": 1533,
  "season": "秋",
  "events": [
    { "type": "harvest", "result": "famine", "koku_delta": -12.3 }
  ]
}
```

### 事実はひとつ、物語は無数

サーバーの状態は全プレイヤーに共通の唯一の真実です。
しかしビューワーは誰でも作れます。歴史書風、武将視点、子供向け、VR没入型。
同じ事実から生まれる物語は無数にあります。

このAPIの設計は、将来的な生成AI連携の自由度を確保しています。サーバー側・クライアント側どちらにAIを組み込むかはビューワーの実装に委ねられており、特定の実装を前提としません。

### 運営のいない戦国

足利幕府（運営）は最初から存在します。しかしそれは歴史の必然です。
プレイヤーが江戸幕府を樹立した瞬間、その時点のルールがオンチェーンに封印され、運営ですら変更できなくなります。シーズンを重ねるごとに、世界は少しずつ分散していきます。

## マルチワールド・マルチシーズン

このエンジンは日本だけを想定していません。

```
第一シーズン   日本・令制国68国（GPS連動）
並行稼働       日本S2 / 世界地図S1 が同時に動く
将来           地球まるごと、任意の地形データで動く
```

### Universe Registry

どのサーバーが「公認」かを管理するオンチェーンのレジストリです。

- 公認サーバーは武将NFTを持ち込める
- フォークされたサーバーは「別の宇宙」として独立して動く
- 競合ではなく共存。Nostrのリレーと同じ考え方

### シーズンの凍結と歴史の永続

シーズンが終わると、その時点の全状態がオンチェーンに封印されます。

```
凍結のプロセス
  シーズン終了条件が満たされる
      ↓
  全状態をスナップショットとしてオンチェーンに記録
      ↓
  サーバーは読み取り専用になる（止める必要はない）
      ↓
  武将カードはスナップショットから次シーズンへ引き継がれる
```

凍結されたシーズンのデータは**特定のサーバーに依存しません**。
オンチェーンに刻まれた記録である以上、元のサーバーが停止しても、
別のサーバーやビューワーから同じ歴史を参照・閲覧・利用できます。

「天文五年の伊豆の石高を見たい」→ 凍結されたシーズン1を参照。
そのサーバーが存在しなくても、歴史は永久に残ります。

## 拡張性

このエンジンはコアロジックと拡張機能を明確に分離しています。
以下は将来的な拡張として設計上の自由度を確保している機能です。現時点での実装を前提としません。

### GPS連動・御朱印帳（将来の拡張）

各令制国の国境ポリゴン（GeoJSON）は実際の日本地図と1:1対応しています。
この構造は将来的なGPS連動の基盤として機能します。

```
現実のGPS座標 → GeoJSONポリゴン判定 → 「あなたは今、伊豆国にいる」
```

御朱印帳アプリ、観光・歴史教育との連携、位置情報ゲームなど、
ビューワーが自由に実装できる余地を残しています。

### 生成AI連携（将来の拡張）

サーバーは事実と数値のみを返します。
どう表現するか、AIをどう使うかはビューワーの実装に完全に委ねられています。
サーバー側・クライアント側どちらへの組み込みも妨げません。

## API（事実のみを返す）

```
GET  /province/{name}/state     国の現在状態
GET  /province/{name}/hex       地形データ
GET  /events/latest             最新イベントログ
GET  /samurai/{id}/status       武将の現在状態
POST /action                    武将の行動（署名必須）
GET  /season/current            現在のシーズン情報
GET  /registry/servers          公認サーバーリスト
```

レスポンスは数値とフラグのみ。文章はゼロ。
どう表現するかはビューワーが自由に決めます。

## 参加する

このプロジェクトはオープンです。

- エンジンのフォークは自由です
- 独自の地形データで独自のサーバーを立てられます
- 公式ビューワー以外のクライアントを作れます
- 令制国以外の世界データを追加できます

**ただし公認サーバーへの参加には Universe Registry への登録が必要です。**
